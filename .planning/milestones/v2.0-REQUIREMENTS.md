## v2.0 Requirements

Requirements for v2.0 milestone (Phases 5-7.1).

### DNS & Ad Blocking

- [x] **DNS-01**: Pi-hole deployed as Docker container on ARM64
- [x] **DNS-02**: Pi-hole web UI accessible at pihole.ragnalab.xyz via Traefik
- [x] **DNS-03**: DNS queries blocked for ad/tracking domains network-wide
- [x] **DNS-04**: Upstream DNS configured (Cloudflare 1.1.1.1 / Quad9 9.9.9.9)

### DHCP

- [~] **DHCP-01**: Pi-hole configured as DHCP server for home network — *DNS-only mode, gateway locked*
- [~] **DHCP-02**: DHCP disabled on Xfinity gateway — *Not possible, gateway locked*
- [x] **DHCP-03**: Pi has static IP (DHCP reservation or static config)
- [~] **DHCP-04**: DHCP hands out Pi-hole as DNS server to all clients — *Manual DNS config required*

### High Availability

- [x] **HA-01**: Fallback DNS configured so internet works if Pi is down
- [x] **HA-02**: DHCP lease time configured for resilience (not too short)
- [x] **HA-03**: Recovery procedure documented

### Observability

- [x] **OBS-01**: Uptime Kuma monitor for Pi-hole DNS (port 53)
- [x] **OBS-02**: Uptime Kuma monitor for Pi-hole web UI
- [x] **OBS-03**: Homepage widget showing Pi-hole statistics (queries blocked, percentage)

### Operations

- [x] **OPS-01**: Pi-hole config included in automated backup

- [x] **OPS-03**: Resource limits configured (memory, CPU)
