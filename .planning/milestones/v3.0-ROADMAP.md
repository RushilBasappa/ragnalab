## Milestone Summary

**Key Decisions:**
- Authelia over Authentik — Lightweight (~30MB), config-file based, better for Pi resources
- Passkeys as 2FA only — Authelia 4.39.14 doesn't support passwordless WebAuthn
- SQLite storage (no Redis) — Sufficient for 4 users, simpler deployment
- Argon2id tuning for ARM64 — m=256, t=1, p=2 prevents slow logins on Pi 5
- Forward-proxy pattern for Dozzle — Native support, reads Remote-User header directly
- Trusted header pattern for Paperless — Django's HTTP_REMOTE_USER for SSO auto-login

**Issues Resolved:**
- WebAuthn config updated for Authelia 4.39.14 (deprecated settings removed)
- ARM64-tuned argon2id parameters to prevent slow password verification
- Post-audit fix: Added SSO middleware to 4 additional services (Bazarr, Maintainerr, Pi-hole, FileBrowser)

**Technical Debt Incurred:**
- Paperless container shows "unhealthy" due to celery worker OOM on Pi (webserver works)
- Session cookie domain 'ragnalab.xyz' vs '.ragnalab.xyz' (minor compatibility concern)

**Deferred to v4.0:**
- Jellyfin SSO plugin (requires plugin install + account linking)
- Jellyseerr OIDC (preview branch stability unknown)
- Vaultwarden OIDC (mobile app 2FA issues)
- Pi-hole auth integration (v6 reverse proxy issues)
