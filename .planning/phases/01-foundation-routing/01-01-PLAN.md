<task type="auto">
  <name>Task 1: Create project structure and Docker networks</name>
  <files>
    scripts/init-networks.sh
    apps/.gitkeep
  </files>
  <action>
1. Create directory structure:
   - `proxy/` - Will hold Traefik and socket proxy infrastructure
   - `proxy/traefik/` - Traefik configuration files
   - `proxy/traefik/dynamic/` - File provider middleware configs
   - `proxy/traefik/acme/` - Let's Encrypt certificate storage
   - `apps/` - Application compose files (one subdirectory per app)
   - `scripts/` - Utility scripts

2. Create `scripts/init-networks.sh`:
```bash
#!/bin/bash
# Create external networks before starting any compose files
set -e

echo "Creating Docker networks..."

docker network create proxy 2>/dev/null && echo "Created 'proxy' network" || echo "Network 'proxy' already exists"
docker network create socket_proxy_network 2>/dev/null && echo "Created 'socket_proxy_network' network" || echo "Network 'socket_proxy_network' already exists"

echo ""
echo "Networks ready:"
docker network ls --filter name=proxy --filter name=socket_proxy_network
```

3. Make script executable: `chmod +x scripts/init-networks.sh`

4. Run the script to create networks: `./scripts/init-networks.sh`

5. Create `apps/.gitkeep` to preserve empty directory in git
  </action>
  <verify>
- `docker network ls | grep -E 'proxy|socket_proxy'` shows both networks
- `ls -la scripts/init-networks.sh` shows executable permissions
- Directory structure exists: proxy/traefik/dynamic/, proxy/traefik/acme/, apps/, scripts/
  </verify>
  <done>
- Docker networks 'proxy' and 'socket_proxy_network' exist and are visible in `docker network ls`
- Project directories created matching research-recommended structure
- Network initialization script is executable and idempotent (can run multiple times safely)
  </done>

<task type="checkpoint:human-action" gate="blocking">
  <name>Task 3: Configure Cloudflare DNS and create API token</name>
  <action>
User must complete these steps in Cloudflare Dashboard:

**Step 1: Create API Token**
1. Go to: https://dash.cloudflare.com/profile/api-tokens
2. Click "Create Token"
3. Use template: "Edit zone DNS"
4. Configure permissions:
   - Zone > Zone > Read
   - Zone > DNS > Edit
5. Zone Resources: Include > Specific zone > ragnalab.xyz
6. Click "Continue to summary" then "Create Token"
7. COPY THE TOKEN NOW (shown only once)

**Step 2: Create Wildcard DNS Record**
1. Go to: https://dash.cloudflare.com -> ragnalab.xyz -> DNS -> Records
2. Click "Add record"
3. Configure:
   - Type: A
   - Name: * (asterisk for wildcard)
   - IPv4 address: YOUR_TAILSCALE_IP (find with `tailscale ip -4`)
   - Proxy status: DNS only (gray cloud, NOT orange)
   - TTL: Auto
4. Click "Save"

**Step 3: Create .env file**
After completing Cloudflare setup, create `proxy/.env`:
```bash
cp proxy/.env.example proxy/.env
# Edit proxy/.env with your actual values
```
  </action>

1. DNS propagation: `dig +short whoami.ragnalab.xyz` should return your Tailscale IP

3. File exists: `test -f proxy/.env && echo "OK" || echo "MISSING"`
