  <files>proxy/traefik/traefik.yml</files>
  <action>
Create `proxy/traefik/traefik.yml` with the following configuration:

```yaml
# Traefik v3.6 Static Configuration
# RagnaLab Homelab Infrastructure

api:
  dashboard: true
  insecure: false  # Dashboard served via routers, not :8080

entryPoints:
  web:
    address: ":80"
    http:
      redirections:
        entryPoint:
          to: websecure
          scheme: https
          permanent: true
  websecure:
    address: ":443"
    http:
      tls: true

providers:
  docker:
    # Connect via socket proxy, NOT direct socket mount
    endpoint: "tcp://socket-proxy:2375"
    exposedByDefault: false
    network: proxy
    watch: true
  file:
    directory: /etc/traefik/dynamic
    watch: true

certificatesResolvers:
  letsencrypt:
    acme:
      email: "${ACME_EMAIL}"
      storage: /etc/traefik/acme/acme.json
      # STAGING SERVER - Switch to production after verification
      # Production: https://acme-v02.api.letsencrypt.org/directory
      caServer: https://acme-staging-v02.api.letsencrypt.org/directory
      dnsChallenge:
        provider: cloudflare
        # Wait for DNS propagation before verification
        delayBeforeCheck: 60
        resolvers:
          - "1.1.1.1:53"
          - "8.8.8.8:53"

log:
  level: INFO

accessLog:
  format: json
  bufferingSize: 100
```

Key design decisions:
- `endpoint: "tcp://socket-proxy:2375"` - Never mount Docker socket directly (SEC-01)
- `exposedByDefault: false` - Services must opt-in with `traefik.enable=true` label
- `caServer: staging` - CRITICAL: Use staging first to avoid rate limits (OPS-02)
- `delayBeforeCheck: 60` - Allow DNS propagation time for Cloudflare
- `insecure: false` - Dashboard only accessible via proper router with TLS

<task type="auto">
  <name>Task 2: Create security middleware configuration</name>
  <files>proxy/traefik/dynamic/middlewares.yml</files>
  <action>
Create `proxy/traefik/dynamic/middlewares.yml`:

```yaml
# Traefik Dynamic Configuration - Middlewares
# Referenced via @file namespace: security-headers@file, rate-limit@file

http:
  middlewares:
    # Security Headers Middleware (SEC-03)
    # Apply to all services: traefik.http.routers.X.middlewares=security-headers@file
    security-headers:
      headers:
        # HSTS - Force HTTPS for 1 year
        stsSeconds: 31536000
        stsIncludeSubdomains: true
        stsPreload: true
        forceSTSHeader: true

        # Prevent clickjacking (X-Frame-Options equivalent)
        frameDeny: true

        # Prevent MIME type sniffing
        contentTypeNosniff: true

        # XSS protection (legacy but still useful)
        browserXssFilter: true

        # Content Security Policy
        # Restrictive default - apps can override with their own middleware
        contentSecurityPolicy: >-
          default-src 'self';
          script-src 'self';
          style-src 'self' 'unsafe-inline';
          img-src 'self' data:;
          font-src 'self';
          object-src 'none';
          base-uri 'self';
          form-action 'self';
          frame-ancestors 'none';

        # Referrer Policy
        referrerPolicy: "strict-origin-when-cross-origin"

        # Permissions Policy (formerly Feature-Policy)
        permissionsPolicy: "geolocation=(), microphone=(), camera=()"

        # Hide server information
        customResponseHeaders:
          server: ""
          x-powered-by: ""

    # Rate Limiting Middleware (SEC-04)
    # Token bucket: 100 requests/second average, burst up to 200
    rate-limit:
      rateLimit:
        average: 100
        burst: 200
        period: 1s

    # Stricter rate limit for sensitive endpoints
    rate-limit-strict:
      rateLimit:
        average: 10
        burst: 20
        period: 1s
```

Design notes:
- `security-headers` covers HSTS, CSP, X-Frame-Options, X-Content-Type-Options, XSS filter
- CSP is restrictive by default - apps needing relaxation can define their own middleware
- `frameDeny: true` is equivalent to X-Frame-Options: DENY
- Two rate limit profiles: standard (100/s) and strict (10/s for auth endpoints)
