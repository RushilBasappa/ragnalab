<task type="auto">
  <name>Task 3: Verify security headers and rate limiting</name>
  <files></files>
  <action>
1. Check security headers on whoami response:
```bash
curl -kI https://whoami.ragnalab.xyz 2>/dev/null | grep -iE "strict-transport|x-frame|x-content-type|content-security|referrer-policy"
```

Expected headers:
- `strict-transport-security: max-age=31536000; includeSubDomains; preload`
- `x-frame-options: DENY` (or equivalent via CSP frame-ancestors)
- `x-content-type-options: nosniff`
- `content-security-policy: ...`
- `referrer-policy: strict-origin-when-cross-origin`

2. Verify server header is hidden:
```bash
curl -kI https://whoami.ragnalab.xyz 2>/dev/null | grep -i "^server:"
# Should be empty or show nothing
```

3. Test rate limiting is active (optional - don't actually exhaust limits):
```bash
# Just verify the middleware is applied in Traefik
docker exec traefik cat /etc/traefik/dynamic/middlewares.yml | grep -q "rate-limit" && echo "Rate limit middleware configured"
```

4. Check dashboard also has security headers:
```bash
curl -kI https://traefik.ragnalab.xyz 2>/dev/null | grep -i "strict-transport-security"
```
  </action>
  <verify>
- `curl -kI https://whoami.ragnalab.xyz 2>/dev/null | grep -qi "strict-transport-security" && echo "HSTS present"`
- `curl -kI https://whoami.ragnalab.xyz 2>/dev/null | grep -qi "x-content-type-options.*nosniff" && echo "X-Content-Type-Options present"`
- `curl -kI https://whoami.ragnalab.xyz 2>/dev/null | grep -qi "content-security-policy" && echo "CSP present"`
  </verify>
  <done>
- Security headers present: HSTS, X-Content-Type-Options, Content-Security-Policy
- Server header is hidden or empty
- Rate limit middleware is loaded in Traefik
- Dashboard also has security headers applied
  </done>
</task>

<task type="auto">
  <name>Task 4: Verify socket proxy security</name>
  <files></files>
  <action>
1. Confirm Traefik uses socket proxy, not direct socket:
```bash
# Traefik container should NOT have docker.sock mounted
docker inspect traefik --format '{{range .Mounts}}{{.Source}} -> {{.Destination}}{{"\n"}}{{end}}' | grep -v docker.sock && echo "No direct socket mount (correct)" || echo "WARNING: Direct socket mount found"
```

2. Confirm socket proxy is read-only:
```bash
# Check environment variables
docker inspect socket-proxy --format '{{range .Config.Env}}{{println .}}{{end}}' | grep "POST=0"
# Should show POST=0
```

3. Test that write operations are blocked (safe test):
```bash
# This should fail because POST=0
docker exec traefik wget -q -O- http://socket-proxy:2375/containers/json 2>&1 && echo "Read works"

# Attempting to create would fail (don't actually run this, just verify POST=0 is set)
```

4. Verify network isolation:
```bash
# Socket proxy should only be on socket_proxy_network
docker inspect socket-proxy --format '{{range $k, $v := .NetworkSettings.Networks}}{{$k}}{{"\n"}}{{end}}'
# Should show: socket_proxy_network (and possibly default)

# Traefik should be on both socket_proxy_network and proxy
docker inspect traefik --format '{{range $k, $v := .NetworkSettings.Networks}}{{$k}}{{"\n"}}{{end}}'
# Should show: socket_proxy_network, proxy
```
  </action>
  <verify>
- `docker inspect traefik --format '{{range .Mounts}}{{.Source}}{{end}}' | grep -v docker.sock | grep -q traefik && echo "Traefik uses config mounts, not socket"`
- `docker inspect socket-proxy --format '{{range .Config.Env}}{{.}}{{end}}' | grep -q "POST=0" && echo "Socket proxy read-only"`
- `docker inspect socket-proxy --format '{{range $k, $v := .NetworkSettings.Networks}}{{$k}}{{end}}' | grep -q "socket_proxy" && echo "Socket proxy on correct network"`
  </verify>
  <done>
- Traefik container does NOT have /var/run/docker.sock mounted
- Socket proxy has POST=0 environment variable
- Socket proxy is only on socket_proxy_network
- Traefik is on both socket_proxy_network and proxy networks
  </done>
</task>
