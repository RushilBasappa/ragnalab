Create the Authelia service stack with SQLite storage (no Redis needed for 4 users):

1. Create directory structure:
   - `stack/infra/authelia/`
   - `stack/infra/authelia/config/`

2. Create `stack/infra/authelia/docker-compose.yml`:
   - Image: `authelia/authelia:4.39.14` (pinned version, ARM64 support)
   - Container name: `authelia`
   - Volumes: `./config:/config`
   - Environment: `TZ=America/Los_Angeles`
   - Network: `proxy`
   - Traefik labels for `auth.ragnalab.xyz` (NO auth middleware - critical!)
   - Autokuma labels for monitoring
   - Homepage labels
   - Resource limits: 128M memory (lightweight)

3. Create `stack/infra/authelia/config/configuration.yml` with:
   - `server.address: 'tcp://:9091'`
   - `log.level: 'debug'` (for initial troubleshooting, reduce later)
   - `theme: 'auto'`
   - `default_2fa_method: 'webauthn'`

   **WebAuthn configuration (CRITICAL - RP ID cannot be changed later):**
   ```yaml
   webauthn:
     disable: false
     display_name: 'RagnaLab'
     rp_id: 'ragnalab.xyz'
     attestation_conveyance_preference: 'indirect'
     user_verification: 'preferred'
   ```

   **Session configuration (CRITICAL - cookie domain for SSO):**
   ```yaml
   session:
     cookies:
       - name: 'authelia_session'
         domain: 'ragnalab.xyz'
         authelia_url: 'https://auth.ragnalab.xyz'
         default_redirection_url: 'https://home.ragnalab.xyz'
         expiration: '1h'
         inactivity: '5m'
         remember_me: '1M'
         same_site: 'lax'
   ```

   **Storage (SQLite - simple, no Redis needed):**
   ```yaml
   storage:
     encryption_key: '${STORAGE_ENCRYPTION_KEY}'
     local:
       path: '/config/db.sqlite3'
   ```

   **Authentication backend (file-based):**
   ```yaml
   authentication_backend:
     file:
       path: '/config/users_database.yml'
       password:
         algorithm: argon2id
         iterations: 1
         memory: 256
         parallelism: 2
         key_length: 32
         salt_length: 16
   ```

   **Access control rules (order matters - first match wins):**
   ```yaml
   access_control:
     default_policy: deny

     rules:
       # 1. Bypass: Plex completely (breaks client apps otherwise)
       - domain: 'plex.ragnalab.xyz'
         policy: bypass

       # 2. Bypass: API endpoints for mobile apps and widgets
       - domain: '*.ragnalab.xyz'
         resources:
           - '^/api(/.*)?$'
           - '^/socket(/.*)?$'
           - '^/System/Info.*'
           - '^/Users/AuthenticateByName.*'
           - '^/notifications(/.*)?$'
           - '^/identity(/.*)?$'
         policy: bypass

       # 3. Admin services (two_factor required) - ACL-01
       - domain:
           - 'traefik.ragnalab.xyz'
           - 'backups.ragnalab.xyz'
           - 'pihole.ragnalab.xyz'
         subject: 'group:admin'
         policy: two_factor

       # 4. Power user services (one_factor) - ACL-02
       - domain:
           - 'sonarr.ragnalab.xyz'
           - 'radarr.ragnalab.xyz'
           - 'prowlarr.ragnalab.xyz'
           - 'bazarr.ragnalab.xyz'
           - 'qbit.ragnalab.xyz'
           - 'maintainerr.ragnalab.xyz'
         subject:
           - 'group:admin'
           - 'group:powerusers'
         policy: one_factor

       # 5. Family services (one_factor) - ACL-03
       - domain:
           - 'jellyfin.ragnalab.xyz'
           - 'requests.ragnalab.xyz'
         subject:
           - 'group:admin'
           - 'group:powerusers'
           - 'group:family'
         policy: one_factor

       # 6. Guest access to Jellyfin only - ACL-05 (partial)
       - domain: 'jellyfin.ragnalab.xyz'
         subject: 'group:guests'
         policy: one_factor

       # 7. General admin access to everything else
       - domain: '*.ragnalab.xyz'
         subject: 'group:admin'
         policy: one_factor
   ```

   **Notifier (filesystem for now - no email needed):**
   ```yaml
   notifier:
     disable_startup_check: false
     filesystem:
       filename: '/config/notification.txt'
   ```

<task type="auto">
  <name>Task 3: Add forwardAuth middleware to Traefik</name>
  <files>
    stack/infra/traefik/config/dynamic/middlewares.yml
  </files>
  <action>
Add the Authelia forwardAuth middleware to Traefik's dynamic configuration.

Update `stack/infra/traefik/config/dynamic/middlewares.yml` to add:

```yaml
    # Authelia ForwardAuth middleware
    # Apply to services: traefik.http.routers.X.middlewares=authelia@file
    authelia:
      forwardAuth:
        address: 'http://authelia:9091/api/authz/forward-auth'
        trustForwardHeader: true
        authResponseHeaders:
          - 'Remote-User'
          - 'Remote-Groups'
          - 'Remote-Email'
          - 'Remote-Name'

    # Authelia with basic auth fallback (for API access if needed)
    authelia-basic:
      forwardAuth:
        address: 'http://authelia:9091/api/verify?auth=basic'
        trustForwardHeader: true
        authResponseHeaders:
          - 'Remote-User'
          - 'Remote-Groups'
```

This defines two middlewares:
- `authelia@file` - Standard forwardAuth for web UI (redirects to login page)
- `authelia-basic@file` - Basic auth prompt for API access (if needed later)

IMPORTANT: Do NOT add authelia middleware to any routers yet - that's Phase 10. This phase only deploys the foundation.
