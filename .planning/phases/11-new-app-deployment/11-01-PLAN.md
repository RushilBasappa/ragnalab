  <name>Task 1: Update Authelia access control rules for new apps</name>
  <files>stack/infra/authelia/config/configuration.yml</files>
  <action>
Add access control rules for the three new apps BEFORE the "General admin access" catch-all rule (rule #7).

Add these two rules after rule #6 (Guest access to Jellyfin):

```yaml
    # 7. Admin services - new apps requiring two_factor
    - domain:
        - 'docs.ragnalab.xyz'
        - 'logs.ragnalab.xyz'
      subject: 'group:admin'
      policy: two_factor

    # 8. Power user services - IT-Tools with one_factor
    - domain: 'tools.ragnalab.xyz'
      subject:
        - 'group:admin'
        - 'group:powerusers'
      policy: one_factor
```

The existing rule #7 (General admin access to everything else) becomes rule #9.

Rationale:
- docs.ragnalab.xyz (Paperless-ngx): Contains sensitive documents, admin-only with 2FA
- logs.ragnalab.xyz (Dozzle): Container logs can expose secrets, admin-only with 2FA
- tools.ragnalab.xyz (IT-Tools): Developer utilities, power users allowed with 1FA
  </action>

<task type="auto">
  <name>Task 2: Create IT-Tools Docker Compose</name>
  <files>stack/apps/it-tools/docker-compose.yml</files>
  <action>
Create directory `stack/apps/it-tools/` and Docker Compose file.

Use this configuration (from research):

```yaml
# stack/apps/it-tools/docker-compose.yml
# IT-Tools - Collection of developer utilities
# Source: https://github.com/CorentinTh/it-tools
# Access at: https://tools.ragnalab.xyz

services:
  it-tools:
    image: corentinth/it-tools:latest
    container_name: it-tools
    restart: unless-stopped
    profiles: ["apps"]
    networks:
      - proxy
    labels:
      # Traefik routing
      - "traefik.enable=true"
      - "traefik.http.routers.it-tools.rule=Host(`tools.ragnalab.xyz`)"
      - "traefik.http.routers.it-tools.entrypoints=websecure"
      - "traefik.http.routers.it-tools.tls=true"
      - "traefik.http.routers.it-tools.tls.certresolver=letsencrypt"
      - "traefik.http.services.it-tools.loadbalancer.server.port=80"
      - "traefik.http.routers.it-tools.middlewares=authelia@file"
      - "traefik.docker.network=proxy"
      # Homepage dashboard
      - "homepage.group=Apps"
      - "homepage.name=IT Tools"
      - "homepage.icon=it-tools.png"
      - "homepage.href=https://tools.ragnalab.xyz"
      - "homepage.description=Developer Utilities"
      - "homepage.server=ragnalab-docker"
      # Autokuma monitoring (internal URL to bypass auth)
      - "kuma.it-tools.http.name=IT Tools"
      - "kuma.it-tools.http.url=http://it-tools:80"
      - "kuma.it-tools.http.parent_name=apps-group"
    deploy:
      resources:
        limits:
          memory: 64M
          cpus: '0.1'
    logging:
      driver: json-file
      options:
        max-size: "10m"
        max-file: "3"

networks:
  proxy:
    external: true
```

Key points:
- No volumes needed (client-side app, preferences in browser localStorage)
- Port 80 (not 8080)
- Uses `apps-group` for Autokuma (already defined in Vaultwarden)
- Resource limits for Pi-friendly deployment

Should show middleware, container name, and hostname.
