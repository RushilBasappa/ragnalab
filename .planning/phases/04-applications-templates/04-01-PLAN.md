  <name>Task 1: Create Homepage docker-compose.yml</name>
  <files>apps/homepage/docker-compose.yml</files>
  <action>
Create apps/homepage/ directory and docker-compose.yml following the established pattern from uptime-kuma.

Required elements:
- Image: ghcr.io/gethomepage/homepage:latest
- Container name: homepage
- Environment: HOMEPAGE_ALLOWED_HOSTS=home.ragnalab.xyz, PUID=1000, PGID=1000
- Volume mounts:
  - ./config:/app/config
  - /var/run/docker.sock:/var/run/docker.sock:ro (direct mount, same pattern as uptime-kuma)
- Network: proxy (external)
- Traefik labels:
  - traefik.enable=true
  - traefik.http.routers.homepage.rule=Host(`home.ragnalab.xyz`)
  - traefik.http.routers.homepage.entrypoints=websecure
  - traefik.http.routers.homepage.tls=true
  - traefik.http.routers.homepage.tls.certresolver=letsencrypt
  - traefik.http.services.homepage.loadbalancer.server.port=3000
  - traefik.docker.network=proxy
- Deploy resource limits (memory: 256M, cpus: 0.5, reservation: 128M)
- Logging config (json-file, max-size 10m, max-file 3)

Note: Homepage config volume is mounted as ./config (local directory), not a named volume, since Homepage config is static YAML files we want in version control.
  </action>
  <verify>cat apps/homepage/docker-compose.yml | grep -q "home.ragnalab.xyz"</verify>
  <done>docker-compose.yml exists with correct Traefik labels and environment variables</done>
