<task type="auto">
  <name>Task 2: Add Homepage labels to infrastructure services</name>
  <files>proxy/docker-compose.yml, apps/uptime-kuma/docker-compose.yml</files>
  <action>
Add Homepage labels to existing infrastructure services so they appear in the dashboard:

1. proxy/docker-compose.yml - Add labels to traefik service:
   - homepage.group=Infrastructure
   - homepage.name=Traefik
   - homepage.icon=traefik.png
   - homepage.href=https://traefik.ragnalab.xyz
   - homepage.description=Reverse Proxy
   - homepage.widget.type=traefik
   - homepage.widget.url=http://traefik:8080

2. apps/uptime-kuma/docker-compose.yml - Add labels:
   - homepage.group=Infrastructure
   - homepage.name=Uptime Kuma
   - homepage.icon=uptime-kuma.png
   - homepage.href=https://status.ragnalab.xyz
   - homepage.description=Service Monitoring
   - homepage.widget.type=uptimekuma
   - homepage.widget.url=http://uptime-kuma:3001
   - homepage.widget.slug=ragnalab (or appropriate status page slug if one exists)

Note: Widget URLs use container names (internal DNS), not external hostnames. This allows Homepage to communicate directly via Docker network.

After adding labels, restart services:
   docker compose -f proxy/docker-compose.yml up -d
   docker compose -f apps/uptime-kuma/docker-compose.yml up -d

Wait 10-15 seconds for Homepage to detect label changes.
  </action>
  <verify>grep -q "homepage.widget.type=traefik" proxy/docker-compose.yml && grep -q "homepage.widget.type=uptimekuma" apps/uptime-kuma/docker-compose.yml</verify>
