<task type="auto">
  <name>Task 2: Add Vaultwarden volume to backup service</name>
  <files>apps/backup/docker-compose.yml</files>
  <action>
Update apps/backup/docker-compose.yml to include Vaultwarden volume:

1. Add volume mount in backup service volumes section:
   - vaultwarden-data:/backup/vaultwarden:ro

2. Update BACKUP_STOP_DURING_BACKUP_LABEL to stop both services:
   - Change from "uptime-kuma" to comma-separated or use label-based discovery
   - Actually, offen/docker-volume-backup v2 uses label-based discovery: any container with docker-volume-backup.stop-during-backup label gets stopped
   - The env var BACKUP_STOP_DURING_BACKUP_LABEL is the label KEY, not value
   - Containers with matching label KEY are stopped regardless of value
   - So: Just add the volume mount, the label-based stop already works

3. Add external volume reference in volumes section:
   - vaultwarden-data:
       external: true
       name: vaultwarden_vaultwarden-data

Note: Volume name format is {compose-project}_{volume-name}. Since the compose file is in apps/vaultwarden/, project name defaults to "vaultwarden", so volume is "vaultwarden_vaultwarden-data".
  </action>
  <verify>grep -q "vaultwarden-data" apps/backup/docker-compose.yml</verify>
  <done>Backup service configured to include Vaultwarden volume</done>
</task>

<task type="auto">
  <name>Task 3: Create Vaultwarden .env and deploy</name>
  <files>apps/vaultwarden/.env</files>
  <action>
1. Generate admin token hash (interactive):
   docker run --rm -it vaultwarden/server /vaultwarden hash
   - Enter a secure admin password when prompted
   - Copy the output hash (starts with $argon2id$...)

2. Create apps/vaultwarden/.env:
   - VAULTWARDEN_ADMIN_TOKEN='<paste-hash-here>'
   - Use single quotes around hash to avoid shell interpretation of $ symbols

3. Start Vaultwarden:
   docker compose -f apps/vaultwarden/docker-compose.yml up -d

4. Wait for container to be ready (15-20 seconds first run for initialization)

5. Verify Traefik discovers the route:
   curl -s http://localhost:8080/api/rawdata | grep -i vaultwarden

6. Test HTTPS access:
   curl -ks https://vault.ragnalab.xyz -o /dev/null -w "%{http_code}"

7. Verify admin panel:
   curl -ks https://vault.ragnalab.xyz/admin -o /dev/null -w "%{http_code}"
   (Should return 200, login page)

8. Restart backup service to pick up new volume:
   docker compose -f apps/backup/docker-compose.yml up -d

If errors:
- Check logs: docker logs vaultwarden
- Verify .env loaded: docker exec vaultwarden env | grep SIGNUPS
  </action>
  <verify>curl -ks https://vault.ragnalab.xyz -o /dev/null -w "%{http_code}" | grep -q "200"</verify>
  <done>Vaultwarden accessible at https://vault.ragnalab.xyz with backup integration active</done>
</task>
