<task type="auto">
  <name>Task 2: Deploy and configure qBittorrent</name>
  <files>apps/media/qbittorrent/docker-compose.yml</files>
  <action>
1. Ensure gluetun is running and healthy:
   docker ps --filter "name=gluetun" --format "{{.Status}}"

2. Start qBittorrent:
   cd /home/rushil/workspace/ragnalab/apps/media/qbittorrent
   docker compose up -d

3. Get initial admin password from logs:
   docker logs qbittorrent 2>&1 | grep -i "temporary password"

   Default username is 'admin'. Note the temporary password.

4. Wait for WebUI to be accessible (may take 10-20 seconds):
   curl -s http://localhost:8080 | head -5

5. CRITICAL - Verify VPN routing:
   docker exec qbittorrent curl -s ifconfig.me

   This MUST show the VPN IP (same as gluetun), NOT your home IP.
   If it shows your home IP, the VPN routing is broken - stop immediately.

6. Log into WebUI (http://localhost:8080):
   - Username: admin
   - Password: from step 3

7. Configure download paths in qBittorrent WebUI:
   Options -> Downloads:
   - Default Save Path: /media/downloads
   - Keep incomplete torrents in: /media/incomplete
   - Enable categories for Sonarr/Radarr integration

8. Change admin password:
   Options -> Web UI -> Change password

9. Configure for arr apps:
   Options -> Downloads:
   - Check "Create subfolder for multi-file torrents"
   Options -> BitTorrent:
   - Disable DHT, PeX, Local Peer Discovery (optional, privacy)
  </action>
  <verify>docker exec qbittorrent curl -s ifconfig.me 2>/dev/null | grep -E '^[0-9]+\.[0-9]+\.[0-9]+\.[0-9]+$'</verify>
  <done>qBittorrent running, VPN routing verified (shows VPN IP), WebUI accessible at localhost:8080</done>
</task>

Alternative via API (if WebUI unavailable):
```bash
# Get SID cookie first

  -b "SID=$SID" -d "category=tv&savePath=/media/downloads/tv"

  -b "SID=$SID" -d "category=movies&savePath=/media/downloads/movies"
```
