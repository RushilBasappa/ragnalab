  <files>apps/media/sonarr/docker-compose.yml, apps/media/radarr/docker-compose.yml</files>
  <action>
Create both docker-compose files following the established pattern.

CRITICAL: Both must mount /media at /media (same path) for hardlinks to work.

**apps/media/sonarr/docker-compose.yml:**
- Image: lscr.io/linuxserver/sonarr:latest
- Container name: sonarr
- Environment: PUID=1000, PGID=1000, TZ=America/Los_Angeles
- Volumes:
  - sonarr-config:/config
  - /media:/media (MUST be identical path)
- Networks: proxy (external)
- Labels:
  Traefik:
  - traefik.enable=true
  - traefik.http.routers.sonarr.rule=Host(`sonarr.ragnalab.xyz`)
  - traefik.http.routers.sonarr.entrypoints=websecure
  - traefik.http.routers.sonarr.tls=true
  - traefik.http.routers.sonarr.tls.certresolver=letsencrypt
  - traefik.http.services.sonarr.loadbalancer.server.port=8989
  - traefik.docker.network=proxy
  Homepage:
  - homepage.group=Media
  - homepage.name=Sonarr
  - homepage.icon=sonarr.png
  - homepage.href=https://sonarr.ragnalab.xyz
  - homepage.description=TV Shows
  - homepage.widget.type=sonarr
  - homepage.widget.url=http://sonarr:8989
  - homepage.widget.key=${SONARR_API_KEY}
  - homepage.server=my-docker
  Backup:
  - docker-volume-backup.stop-during-backup=sonarr
- Resource limits: memory 512M, cpus 0.5, reservation 256M
- Logging: json-file, max-size 10m, max-file 3
- restart: unless-stopped

**apps/media/radarr/docker-compose.yml:**
- Image: lscr.io/linuxserver/radarr:latest
- Container name: radarr
- Same environment, volumes, network pattern as Sonarr
- Traefik: radarr.ragnalab.xyz, port 7878
- Homepage: Movies description, radarr widget type
- Same resource limits and logging

Include volumes and networks sections at bottom of each file.

<task type="auto">
  <name>Task 2: Deploy and configure Sonarr</name>
  <files>apps/media/sonarr/docker-compose.yml</files>
  <action>
1. Start Sonarr:
   cd /home/rushil/workspace/ragnalab/apps/media/sonarr
   docker compose up -d

2. Wait for startup (20-30 seconds):
   docker logs sonarr 2>&1 | tail -5

3. Verify Traefik route:
   curl -ks https://sonarr.ragnalab.xyz -o /dev/null -w "%{http_code}"

4. Access Sonarr at https://sonarr.ragnalab.xyz

5. Configure authentication:
   Settings -> General -> Security
   - Authentication: Forms (Login Page)
   - Set Username and Password

6. Get API key:
   Settings -> General -> Security -> API Key
   Add to apps/media/.env: SONARR_API_KEY=your_key

7. Add Download Client:
   Settings -> Download Clients -> Add -> qBittorrent
   - Host: gluetun (qBittorrent uses gluetun's network)
   - Port: 8080
   - Username: admin
   - Password: (your qBittorrent password)
   - Category: tv
   - Test connection before saving

8. Configure Root Folder:
   Settings -> Media Management -> Root Folders -> Add Root Folder
   - Path: /media/library/tv
   This is where organized TV shows will be stored.

9. Connect to Prowlarr (Prowlarr side):
   In Prowlarr: Settings -> Apps -> Add -> Sonarr
   - Prowlarr Server: http://prowlarr:9696
   - Sonarr Server: http://sonarr:8989
   - API Key: (Sonarr API key from step 6)
   - Sync Categories appropriately
   - Test and Save
  </action>
  <verify>curl -ks https://sonarr.ragnalab.xyz -o /dev/null -w "%{http_code}" | grep -q "200\|401\|302"</verify>
  <done>Sonarr accessible, connected to qBittorrent, synced with Prowlarr</done>
</task>
