  <name>Task 1: Create media directory structure</name>
  <files>/media/downloads/movies, /media/downloads/tv, /media/library/movies, /media/library/tv, /media/incomplete</files>
  <action>
Create the media directory structure that enables hardlinks between downloads and library.

CRITICAL: All directories must be on the same filesystem for hardlinks to work.

Directory structure:
```
/media/
├── downloads/           # qBittorrent complete folder
│   ├── movies/          # Radarr category
│   └── tv/              # Sonarr category
├── library/             # Final media location
│   ├── movies/          # Radarr root folder
│   └── tv/              # Sonarr root folder
└── incomplete/          # qBittorrent temp folder
```

Commands:
```bash
sudo mkdir -p /media/{downloads/{movies,tv},library/{movies,tv},incomplete}
sudo chown -R 1000:1000 /media
sudo chmod -R 775 /media
```

Verify ownership matches PUID/PGID (1000:1000) that all media containers will use.
  </action>
  <verify>ls -la /media/ && ls -la /media/downloads/ && ls -la /media/library/</verify>
  <done>All media directories exist with 1000:1000 ownership and 775 permissions</done>
</task>

<task type="auto">
  <name>Task 2: Create Gluetun docker-compose and environment template</name>
  <files>apps/media/gluetun/docker-compose.yml, apps/media/.env.example</files>
  <action>
Create apps/media/ directory and Gluetun configuration.

1. Create apps/media/.env.example with placeholder values:
```env
# VPN Configuration
# Provider options: protonvpn, mullvad, nordvpn, pia, surfshark, etc.
# See: https://github.com/qdm12/gluetun-wiki/tree/main/setup/providers
VPN_SERVICE_PROVIDER=protonvpn

# WireGuard credentials from VPN provider dashboard
# ProtonVPN: account.protonvpn.com -> Downloads -> WireGuard configuration
WIREGUARD_PRIVATE_KEY=your_private_key_here
WIREGUARD_ADDRESSES=10.2.0.2/32

# Optional: Server location preference
SERVER_COUNTRIES=United States

# API Keys (populated after services start)
SONARR_API_KEY=
RADARR_API_KEY=
PROWLARR_API_KEY=
JELLYFIN_API_KEY=
```

2. Create apps/media/gluetun/docker-compose.yml:
- Image: qmcgaw/gluetun:latest
- Container name: gluetun
- cap_add: NET_ADMIN (required for VPN)
- Devices: /dev/net/tun (TUN device for VPN)
- Environment variables from .env (use ${VAR} syntax)
- VPN_TYPE: wireguard (faster than OpenVPN on Pi)
- TZ: America/Los_Angeles
- Ports exposed through gluetun (qBittorrent WebUI 8080)
- Named volume: gluetun-data for /gluetun
- Resource limits: memory 128M, cpus 0.25
- Logging: json-file, max-size 10m, max-file 3
- restart: unless-stopped

Note: Gluetun has built-in healthcheck (queries http://127.0.0.1:9999/) - no custom healthcheck needed.
  </action>
  <verify>cat apps/media/gluetun/docker-compose.yml | grep -q "qmcgaw/gluetun" && cat apps/media/.env.example | grep -q "WIREGUARD_PRIVATE_KEY"</verify>
  <done>Gluetun docker-compose.yml and .env.example created with correct configuration</done>
</task>

<task type="checkpoint:human-action" gate="blocking">
  <what-needed>VPN credentials from provider dashboard</what-needed>
  <instructions>
Before Gluetun can start, you need WireGuard credentials from your VPN provider.

For ProtonVPN (recommended):
1. Go to https://account.protonvpn.com/downloads
2. Click "WireGuard configuration"
3. Select a server location (e.g., US)
4. Copy the private key and address

For other providers, see: https://github.com/qdm12/gluetun-wiki/tree/main/setup/providers

Then:
1. Copy the example env file:
   cp apps/media/.env.example apps/media/.env

2. Edit apps/media/.env with your credentials:
   - VPN_SERVICE_PROVIDER: your provider name
   - WIREGUARD_PRIVATE_KEY: your private key
   - WIREGUARD_ADDRESSES: your assigned IP (e.g., 10.2.0.2/32)
  </instructions>
  <resume-signal>Type "credentials configured" when .env file is ready</resume-signal>
</task>

<task type="auto">
  <name>Task 3: Deploy and verify Gluetun VPN</name>
  <files>apps/media/gluetun/docker-compose.yml</files>
  <action>
1. Start Gluetun with the configured credentials:
   cd /home/rushil/workspace/ragnalab/apps/media/gluetun
   docker compose --env-file ../. up -d

2. Wait for healthcheck (up to 30 seconds):
   docker inspect gluetun --format='{{.State.Health.Status}}'

3. Verify VPN connection:
   docker exec gluetun wget -qO- ifconfig.me

   This should show a VPN IP, NOT your home IP.

4. Check container logs for connection status:
   docker logs gluetun | grep -i "vpn"

5. Verify healthcheck is passing:
   docker inspect gluetun --format='{{json .State.Health}}' | jq

If VPN fails to connect:
- Check credentials in .env match provider dashboard exactly
- Verify VPN_SERVICE_PROVIDER spelling matches gluetun documentation
- Check logs: docker logs gluetun
  </action>

  <done>Gluetun container healthy and showing VPN IP address (not home IP)</done>
