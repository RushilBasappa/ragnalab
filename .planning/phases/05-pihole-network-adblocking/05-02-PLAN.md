<objective>
Configure Pi-hole as the DHCP server for the home network, replacing the Xfinity gateway's DHCP. This ensures all devices automatically use Pi-hole for DNS without any client configuration.

Purpose: Xfinity gateway has locked DNS settings - the only way to provide Pi-hole DNS to all network devices is to have Pi-hole control DHCP. This also enables client identification in Pi-hole's query log.

Output: All network devices receive Pi-hole as their DNS server via DHCP, with fallback DNS configured for resilience.
</objective>

<task type="auto">
  <name>Task 1: Create DHCP configuration with fallback DNS</name>
  <files>apps/pihole/etc-dnsmasq.d/05-custom-dhcp.conf</files>
  <action>
Create dnsmasq configuration file for Pi-hole's DHCP server.

**apps/pihole/etc-dnsmasq.d/05-custom-dhcp.conf:**
```conf
# Pi-hole DHCP Configuration
# Network: 10.0.0.0/24

# DHCP range: .100 to .199, 24 hour leases
# Leaves .1-.99 for static devices, .200-.254 for infrastructure
dhcp-range=10.0.0.100,10.0.0.199,24h

# DNS servers: Pi-hole primary, Cloudflare fallback
# Note: Some clients may use either randomly, not as strict failover
# This is acceptable - primary goal is resilience if Pi-hole is down
dhcp-option=option:dns-server,10.0.0.200,1.1.1.1

# Gateway
dhcp-option=option:router,10.0.0.1

# Domain name
dhcp-option=option:domain-name,local

# Raspberry Pi static reservation (ensure Pi always gets same IP)
# Get MAC address: ip link show eth0 | grep ether
dhcp-host=2c:cf:67:93:77:67,ragnalab-pi,10.0.0.245

# Pi-hole container static reservation
# Ensures macvlan IP doesn't conflict with DHCP pool
dhcp-host=id:*,set:pihole,ignore
```

The configuration:
- Allocates .100-.199 for dynamic clients (100 addresses)
- Provides Pi-hole as primary DNS with 1.1.1.1 as fallback
- Sets 24-hour lease time for stability
- Reserves Raspberry Pi's current IP (10.0.0.245)
- Points to Xfinity gateway (10.0.0.1) as router

After creating file, restart Pi-hole container to load config:
```bash
cd apps/pihole
docker compose restart pihole
```

Wait for container to restart, then verify config loaded:
```bash
docker exec pihole cat /etc/dnsmasq.d/05-custom-dhcp.conf
```

<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Disable Xfinity Gateway DHCP</name>
  <what>
DHCP must be disabled on the Xfinity gateway to prevent conflicts. Two DHCP servers on the same network causes IP conflicts and network instability.
  </what>
  <instructions>
**Before proceeding:** This will temporarily disrupt network for devices as they renew leases. Best done when few devices are active.

1. Open browser to http://10.0.0.1
2. Log in with Xfinity gateway credentials
3. Navigate to: Gateway > Connection > Local IP Network (or similar - varies by model)
4. Find DHCP settings section
5. Set DHCP to "Disabled" or "Off"
6. Save/Apply changes

**Document the original DHCP settings before disabling** (IP range, lease time) in case rollback is needed.

**If you cannot find DHCP settings:**
- XB6/XB7/XB8 gateways: Settings > Network > Advanced Settings
- Older models: Gateway > Connection > Local IP Network
- Some models: May require bridge mode or calling Xfinity support
  </instructions>
  <resume-signal>Type "dhcp disabled" when Xfinity gateway DHCP is off, or describe any issues</resume-signal>
</task>
