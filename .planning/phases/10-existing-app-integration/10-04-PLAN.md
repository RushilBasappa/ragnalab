<objective>
Configure qBittorrent with reverse proxy settings and IP whitelist, then add Authelia middleware to the gluetun container (which handles qBittorrent's Traefik routing).

Purpose: qBittorrent runs inside the gluetun VPN container and sees the Docker network IP of Traefik, not the real client IP. The reverse proxy settings tell qBittorrent to trust the X-Forwarded-For header and whitelist the Docker network CIDR to bypass qBittorrent's built-in auth. Authelia then handles authentication.

Output: qBittorrent.conf updated with reverse proxy settings. Gluetun docker-compose.yml updated with authelia middleware label.
</objective>

<task type="auto">
  <name>Task 1: Configure qBittorrent reverse proxy settings</name>
  <files>qBittorrent.conf (in Docker volume)</files>
  <action>
1. Stop the qBittorrent container (and gluetun since qBittorrent depends on it):
   ```bash
   docker stop port-manager qbittorrent gluetun
   ```

2. Locate qBittorrent.conf in the Docker volume:
   ```bash
   # The config file is at: /config/qBittorrent/qBittorrent.conf inside the volume
   # Volume: ragnalab_qbittorrent-config
   CONFIG_PATH="/var/lib/docker/volumes/ragnalab_qbittorrent-config/_data/qBittorrent/qBittorrent.conf"

   # Verify file exists
   sudo ls -la $CONFIG_PATH
   ```

3. Add/update the reverse proxy settings in qBittorrent.conf:

   These settings must be in the `[Preferences]` section:
   ```ini
   WebUI\ReverseProxySupportEnabled=true
   WebUI\TrustedReverseProxiesList=172.0.0.0/8
   WebUI\AuthSubnetWhitelistEnabled=true
   WebUI\AuthSubnetWhitelist=172.0.0.0/8
   WebUI\CSRFProtection=false
   WebUI\HostHeaderValidation=false
   WebUI\MaxAuthenticationFailCount=0
   ```

   Use this command to add the settings (if not present) or update them:
   ```bash
   CONFIG_PATH="/var/lib/docker/volumes/ragnalab_qbittorrent-config/_data/qBittorrent/qBittorrent.conf"

   # Backup the config first
   sudo cp $CONFIG_PATH ${CONFIG_PATH}.bak

   # Check if settings already exist and update/add as needed
   # Remove any existing reverse proxy settings to avoid duplicates
   sudo sed -i '/WebUI\\ReverseProxySupportEnabled/d' $CONFIG_PATH
   sudo sed -i '/WebUI\\TrustedReverseProxiesList/d' $CONFIG_PATH
   sudo sed -i '/WebUI\\AuthSubnetWhitelistEnabled/d' $CONFIG_PATH
   sudo sed -i '/WebUI\\AuthSubnetWhitelist=/d' $CONFIG_PATH
   sudo sed -i '/WebUI\\CSRFProtection/d' $CONFIG_PATH
   sudo sed -i '/WebUI\\HostHeaderValidation/d' $CONFIG_PATH
   sudo sed -i '/WebUI\\MaxAuthenticationFailCount/d' $CONFIG_PATH

   # Add the settings at the end of the [Preferences] section
   # First, find if [Preferences] section exists
   if sudo grep -q "^\[Preferences\]" $CONFIG_PATH; then
     # Add settings after [Preferences] line
     sudo sed -i '/^\[Preferences\]/a WebUI\\ReverseProxySupportEnabled=true\nWebUI\\TrustedReverseProxiesList=172.0.0.0/8\nWebUI\\AuthSubnetWhitelistEnabled=true\nWebUI\\AuthSubnetWhitelist=172.0.0.0/8\nWebUI\\CSRFProtection=false\nWebUI\\HostHeaderValidation=false\nWebUI\\MaxAuthenticationFailCount=0' $CONFIG_PATH
   else
     # Add [Preferences] section with settings
     echo -e "\n[Preferences]\nWebUI\\ReverseProxySupportEnabled=true\nWebUI\\TrustedReverseProxiesList=172.0.0.0/8\nWebUI\\AuthSubnetWhitelistEnabled=true\nWebUI\\AuthSubnetWhitelist=172.0.0.0/8\nWebUI\\CSRFProtection=false\nWebUI\\HostHeaderValidation=false\nWebUI\\MaxAuthenticationFailCount=0" | sudo tee -a $CONFIG_PATH
   fi
   ```

4. Verify the settings:
   ```bash
   sudo grep -A10 "^\[Preferences\]" $CONFIG_PATH | head -15
   ```
  </action>

<task type="auto">
  <name>Task 2: Add Authelia middleware to gluetun (qBittorrent router)</name>
  <files>stack/media/qbittorrent/docker-compose.yml</files>
  <action>
Add the Authelia ForwardAuth middleware label to the gluetun service's qBittorrent router.

Note: The qBittorrent Traefik labels are on the gluetun service (not qbittorrent) because qBittorrent uses gluetun's network stack.

In the gluetun service's labels section, add this line after the existing qBittorrent router labels:
```yaml
- "traefik.http.routers.qbittorrent.middlewares=authelia@file"
```

Place it after the `traefik.http.services.qbittorrent.loadbalancer.server.port=8080` line and before the `traefik.docker.network=proxy` line.
  </action>

  </verify>
  <done>gluetun docker-compose.yml has authelia middleware label for qBittorrent router</done>
</task>
