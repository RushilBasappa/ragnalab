<objective>
Protect Uptime Kuma and Backrest with Authelia SSO, disabling their built-in authentication to avoid double login.

Purpose: These services have their own login pages. Without disabling built-in auth, users would authenticate twice (once at Authelia, once at the app). This plan removes the app-level auth and relies solely on Authelia.

Output: Two docker-compose.yml files updated with authelia@file middleware. Both services' built-in auth disabled via their respective UIs.
</objective>

<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Disable Uptime Kuma built-in authentication</name>
  <action>
User must disable Uptime Kuma's built-in authentication via the web UI.

This step CANNOT be automated because:
- Uptime Kuma stores auth settings in its SQLite database
- No CLI or API exists to disable authentication
- The setting requires current password confirmation
  </action>
  <instructions>
1. Open https://status.ragnalab.xyz in your browser
2. Log in with your current Uptime Kuma credentials
3. Navigate to: Settings (gear icon) -> Security
4. Click "Disable Auth"
5. Enter your current password to confirm
6. Click "Disable Auth" button

After this, Uptime Kuma will be accessible without login (temporarily vulnerable until middleware is added).
  </instructions>
  <resume-signal>Type "disabled" when Uptime Kuma auth is disabled</resume-signal>
</task>

<task type="auto">
  <name>Task 2: Add Authelia middleware to Uptime Kuma</name>
  <files>stack/infra/uptime-kuma/docker-compose.yml</files>
  <action>
Add the Authelia ForwardAuth middleware label to the Uptime Kuma service.

In the labels section, add this line after the existing Traefik router labels:
```yaml
- "traefik.http.routers.uptime-kuma.middlewares=authelia@file"
```

Place it after the `traefik.http.services.uptime-kuma.loadbalancer.server.port=3001` line and before the `traefik.docker.network=proxy` line.
  </action>
  <verify>
Run `grep "middlewares=authelia" stack/infra/uptime-kuma/docker-compose.yml` to confirm middleware label is present.
  </verify>
  <done>Uptime Kuma docker-compose.yml contains authelia middleware label</done>
</task>
