<task type="auto">
  <name>Task 1: Migrate from Let's Encrypt staging to production</name>
  <files>proxy/traefik/traefik.yml, proxy/traefik/acme/acme.json</files>
  <action>
Migrate Let's Encrypt from staging to production certificates:

1. Stop Traefik to release certificate lock:
```bash
docker compose -f proxy/docker-compose.yml stop traefik
```

2. Backup and remove staging certificates:
```bash
# Backup staging certificates
cp proxy/traefik/acme/acme.json proxy/traefik/acme/acme-staging-backup.json 2>/dev/null || true

# Remove staging certificates (Traefik won't request new certs if old ones exist)
rm -f proxy/traefik/acme/acme.json
touch proxy/traefik/acme/acme.json
chmod 600 proxy/traefik/acme/acme.json
```

3. Update traefik.yml - comment out or remove the staging caServer line:
```yaml
certificatesResolvers:
  letsencrypt:
    acme:
      email: "rushil.basappa@gmail.com"
      storage: /etc/traefik/acme/acme.json
      # PRODUCTION - Remove or comment out staging server
      # caServer: https://acme-staging-v02.api.letsencrypt.org/directory
      dnsChallenge:
        provider: cloudflare
        ...
```

When caServer is omitted, Traefik defaults to production: https://acme-v02.api.letsencrypt.org/directory

4. Start Traefik:
```bash
docker compose -f proxy/docker-compose.yml start traefik
```

5. Watch logs for certificate issuance:
```bash
docker logs -f traefik 2>&1 | grep -E "(acme|certificate)" --line-buffered &
sleep 90  # DNS-01 challenge takes ~60s
kill %1 2>/dev/null
```

<task type="auto">
  <name>Task 2: Add resource limits to all containers</name>
  <files>proxy/docker-compose.yml, apps/whoami/docker-compose.yml</files>
  <action>
Add deploy.resources to all containers in proxy/docker-compose.yml and apps/whoami/docker-compose.yml.

Resource allocation strategy for Pi 5 (8GB RAM, 4 cores):
- Tailscale: 128M memory, 0.25 CPU (lightweight VPN)
- Socket-proxy: 64M memory, 0.1 CPU (minimal API proxy)
- Traefik: 256M memory, 0.5 CPU (handles all routing)
- Whoami: 64M memory, 0.25 CPU (test service)

Add to each service in docker-compose files:

```yaml
services:
  servicename:
    ...
    deploy:
      resources:
        limits:
          memory: 128M
          cpus: '0.25'
        reservations:
          memory: 64M
```

Also ensure all services have `restart: unless-stopped` for OPS-04.

Verify existing containers don't exceed these limits:
```bash
docker stats --no-stream
```
  </action>
  <verify>
Run: `docker compose -f proxy/docker-compose.yml config 2>&1 | grep -A5 "deploy:" | head -20`
Expected: Shows memory and cpu limits for services
  </verify>
  <done>Container resource limits configured</done>
</task>
