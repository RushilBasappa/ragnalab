<task type="auto">
  <name>Task 1: Enable IP forwarding for Tailscale subnet routing</name>
  <files>/etc/sysctl.d/99-tailscale.conf</files>
  <action>
Create sysctl configuration to enable IPv4 and IPv6 forwarding for Tailscale subnet routing:

```bash
# Create sysctl config
echo 'net.ipv4.ip_forward = 1' | sudo tee /etc/sysctl.d/99-tailscale.conf
echo 'net.ipv6.conf.all.forwarding = 1' | sudo tee -a /etc/sysctl.d/99-tailscale.conf

# Apply immediately
sudo sysctl -p /etc/sysctl.d/99-tailscale.conf
```

This enables the Pi to route traffic between Tailscale clients and Docker containers.
  </action>
  <verify>
Run: `sysctl net.ipv4.ip_forward`
Expected: `net.ipv4.ip_forward = 1`
  </verify>
  <done>IP forwarding enabled and persists across reboots</done>
</task>

<task type="auto">
  <name>Task 2: Enable cgroup memory support for Docker resource limits</name>
  <files>/boot/firmware/cmdline.txt</files>
  <action>
Check if cgroup memory is already enabled:

```bash
docker info 2>&1 | grep -i "memory"
```

If output shows "WARNING: No memory limit support", add cgroup parameters to boot config:

```bash
# Backup existing config
sudo cp /boot/firmware/cmdline.txt /boot/firmware/cmdline.txt.bak

# Add cgroup parameters (append to existing line, do NOT add newline)
PARAMS="cgroup_enable=memory swapaccount=1 cgroup_memory=1 cgroup_enable=cpuset"
if ! grep -q "cgroup_enable=memory" /boot/firmware/cmdline.txt; then
  sudo sed -i "s/$/ $PARAMS/" /boot/firmware/cmdline.txt
  echo "Cgroup parameters added. Reboot required."
else
  echo "Cgroup parameters already configured."
fi
```

IMPORTANT: The cmdline.txt must remain a single line. Do not add newlines.
  </action>
  <verify>
Run: `grep cgroup_enable=memory /boot/firmware/cmdline.txt`
Expected: Line contains `cgroup_enable=memory`
  </verify>
  <done>Kernel boot parameters configured for Docker memory limits</done>
</task>

<task type="auto">
  <name>Task 3: Verify thermal status and active cooling</name>
  <files>None (read-only verification)</files>
  <action>
Check Pi 5 thermal status and throttling history:

```bash
# Current temperature
vcgencmd measure_temp

# Throttling status (0x0 = no throttling)
vcgencmd get_throttled

# Alternative temperature reading
cat /sys/class/thermal/thermal_zone0/temp
```

Acceptable ranges:
- Temperature: Under 70C at idle is healthy
- Throttling: 0x0 means no current or past throttling
- Any non-zero throttle value indicates cooling issues

If temperature >70C at idle or throttling detected, note this as a concern but continue - active cooling can be addressed separately.
  </action>
  <verify>
Run: `vcgencmd measure_temp && vcgencmd get_throttled`
Expected: Temperature under 80C, throttled=0x0
  </verify>
  <done>Thermal status documented, no blocking issues</done>
</task>
