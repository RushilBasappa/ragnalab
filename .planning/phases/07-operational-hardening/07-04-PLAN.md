  <name>Task 1: Extend socket-proxy permissions for all consumers</name>
  <files>
    - stack/infra/socket-proxy/docker-compose.yml
  </files>
  <action>
Update socket-proxy environment to support all consumers (Traefik, Homepage, Uptime Kuma):

Required permissions (union of all consumer needs):
```yaml
environment:
  # Required: Traefik (discovery), Homepage (status), Uptime Kuma (monitors)
  - CONTAINERS=1
  # Required: Traefik (routing)
  - NETWORKS=1
  # Required: Homepage (shows image versions) - enable this
  - IMAGES=1
  # Optional: System info endpoints - enable for Homepage
  - INFO=1
  # Events for real-time updates (keep enabled)
  - EVENTS=1
  # Deny all write operations (unchanged)
  - POST=0
  - BUILD=0
  - COMMIT=0
  - CONFIGS=0
  - SECRETS=0
  - VOLUMES=0
  - AUTH=0
  - EXEC=0
  - PLUGINS=0
  - SWARM=0
  - SYSTEM=0
  - LOG_LEVEL=info
```

The key changes from current config:
- `IMAGES=0` -> `IMAGES=1` (Homepage needs this for version display)
- `INFO=1` added (optional but useful for Homepage system info)

All other settings remain the same. No write operations enabled.
  </action>

<task type="auto">
  <name>Task 2: Migrate Uptime Kuma to socket-proxy</name>
  <files>
    - stack/infra/uptime-kuma/docker-compose.yml
  </files>
  <action>
Update Uptime Kuma compose to use socket-proxy instead of direct docker.sock:

1. Remove the docker.sock volume mount:
   ```yaml
   # DELETE THIS LINE:
   # - /var/run/docker.sock:/var/run/docker.sock:ro
   ```

2. Add socket_proxy_network to networks:
   ```yaml
   networks:
     - proxy
     - socket_proxy_network
   ```

3. Restart Uptime Kuma:
   ```bash
   docker compose --profile infra up -d uptime-kuma
   ```

4. **CRITICAL - UI Configuration Required:**
   After container restarts, Uptime Kuma needs Docker host configuration via UI:
   - Go to https://status.ragnalab.xyz
   - Navigate to Settings > Docker Hosts
   - Add new host:
     - Name: `my-docker`
     - Connection Type: `TCP / HTTP`
     - Docker Host: `socket-proxy`
     - Docker Port: `2375`
   - Click "Test" to verify connection
   - Save

5. Update any existing Docker container monitors to use the new `my-docker` host.

Note: Uptime Kuma stores Docker host config in its database, not environment variables. There is no DOCKER_HOST env var support - UI config is required.
  </action>
  <verify>
```bash

# Should return 200
```

<task type="auto">
  <name>Task 3: Migrate Homepage to socket-proxy</name>
  <files>
    - stack/infra/homepage/docker-compose.yml
    - stack/infra/homepage/config/docker.yaml
  </files>
  <action>
Update Homepage to use socket-proxy:

1. Update docker-compose.yml:
   - Remove docker.sock volume mount
   - Add socket_proxy_network to networks
   - Remove PGID=123 (docker group not needed without socket)

   ```yaml
   services:
     homepage:
       # ... existing config ...
       environment:
         - HOMEPAGE_ALLOWED_HOSTS=home.ragnalab.xyz
         - PUID=1000
         # REMOVE: - PGID=123
       volumes:
         - ./config:/app/config
         # REMOVE: - /var/run/docker.sock:/var/run/docker.sock:ro
       networks:
         - proxy
         - socket_proxy_network  # ADD THIS
   ```

2. Update config/docker.yaml to use TCP connection:
   ```yaml
   # Docker Socket Configuration via socket-proxy
   # https://gethomepage.dev/configs/docker/

   my-docker:
     host: socket-proxy
     port: 2375
   ```

3. Restart Homepage:
   ```bash
   docker compose --profile infra up -d homepage
   ```

4. Verify Homepage can discover containers and show status.
  </action>
