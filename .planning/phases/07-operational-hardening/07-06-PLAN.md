Create Autokuma service in stack/infra/autokuma/:

1. Create stack/infra/autokuma/docker-compose.yml:
```yaml
# Autokuma - Automated Uptime Kuma Monitor Creation
# Creates monitors from Docker labels automatically

services:
  autokuma:
    image: ghcr.io/bigboot/autokuma:master
    container_name: autokuma
    restart: unless-stopped
    profiles: ["infra"]
    depends_on:
      - uptime-kuma
    environment:
      # Uptime Kuma connection
      - AUTOKUMA__KUMA__URL=http://uptime-kuma:3001
      - AUTOKUMA__KUMA__USERNAME=${UPTIME_KUMA_USERNAME}
      - AUTOKUMA__KUMA__PASSWORD=${UPTIME_KUMA_PASSWORD}
      # Tag for tracking managed monitors
      - AUTOKUMA__TAG_NAME=autokuma
      - AUTOKUMA__TAG_COLOR=#4287f5
      # Docker socket for label discovery
      - AUTOKUMA__DOCKER__HOSTS=tcp://socket-proxy:2375
      # Delete monitors when labels removed (with 5min grace period)
      - AUTOKUMA__ON_DELETE=delete
      - AUTOKUMA__DELETE_GRACE_PERIOD=300
      # Default settings for all monitors
      - AUTOKUMA__DEFAULT_SETTINGS={"interval": 60, "maxretries": 3, "retryInterval": 30}
    volumes:
      - autokuma-data:/data
    networks:
      - socket_proxy_network
      - proxy
    deploy:
      resources:
        limits:
          memory: 128M
          cpus: '0.25'
        reservations:
          memory: 64M
    logging:
      driver: json-file
      options:
        max-size: "10m"
        max-file: "3"

volumes:
  autokuma-data:

networks:
  socket_proxy_network:
    external: true
  proxy:
    external: true
```

2. Add to stack/infra/docker-compose.yml includes:
```yaml
include:
  # ... existing includes ...
  - path: autokuma/docker-compose.yml
```

3. Add required environment variables to stack/infra/.env (or root .env):
```
UPTIME_KUMA_USERNAME=<your-uptime-kuma-username>
UPTIME_KUMA_PASSWORD=<your-uptime-kuma-password>
```

Note: These credentials must match your Uptime Kuma login.

4. Before starting Autokuma, delete existing manual monitors in Uptime Kuma UI:
   - Go to https://status.ragnalab.xyz
   - Delete all existing monitors (Autokuma will recreate from labels)
   - This provides a clean slate as per user decision

5. Start Autokuma:
```bash
docker compose --profile infra up -d autokuma
```

  <action>
Add kuma labels to each infrastructure service for automatic monitoring.

**Label format:** `kuma.<id>.<type>.<setting>: <value>`

**Monitor types:**
- `http`: HTTP(S) endpoint monitoring
- `docker`: Docker container status monitoring

**Parent groups:** Use parent_name to organize monitors:
- Infrastructure (for HTTP monitors)
- Containers (for container monitors)

Add labels to each service:

1. **Traefik** (stack/infra/traefik/docker-compose.yml):
```yaml
labels:
  # ... existing labels ...
  # Autokuma monitoring
  - "kuma.traefik.http.name=Traefik"
  - "kuma.traefik.http.url=https://traefik.ragnalab.xyz"
  - "kuma.traefik.http.parent_name=Infrastructure"
  - "kuma.traefik-container.docker.name=Traefik Container"
  - "kuma.traefik-container.docker.docker_container=traefik"
  - "kuma.traefik-container.docker.docker_host=my-docker"
  - "kuma.traefik-container.docker.parent_name=Containers"
```
